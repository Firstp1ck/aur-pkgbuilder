# wiki-update

Update the GitHub wiki extensively with the changes on the current
branch.

- Keep it user-friendly, clear, concise, and complete.
- Add dedicated sections only when the change does not fit under an
  existing one — prefer weaving new content into the current structure.
- Keep feature descriptions in sync with `README.md`; the wiki is the
  long-form companion, not a replacement for the top-level docs.
- When a new navigation page or config field lands, update both the
  "Usage" and "Configuration" wiki pages (create them if missing).

If the wiki checkout isn't available locally yet, clone it alongside
the main repo:

```bash
git clone https://github.com/Firstp1ck/aur-pkgbuilder.wiki.git
```
